# 프로젝트 받아오기 — 코드를 내 컴퓨터로 가져오기

이제 GitHub에 올라와 있는 HRA 프로젝트 코드를 내 컴퓨터로 통째로 가져오는 방법을 알아볼게요. 이 과정을 'Clone(클론)'이라고 불러요.

---

### 1. 작업 폴더 만들기
작업할 공간을 정해야 해요. 바탕화면(Desktop)에서 시작해볼게요.

=== "명령어 입력"
    터미널을 열고 다음을 입력해서 바탕화면으로 이동해요.
    ```bash title="바탕화면으로 이동"
    cd Desktop
    ```

---

### 2. Git Clone (프로젝트 복제)
GitHub에서 폴더를 다운로드받는 명령어예요. 복사해서 터미널에 붙여넣으세요.

```bash title="프로젝트 복제"
git clone https://github.com/yeongseon/hra-website.git
```
!!! info "이게 뭔가요?"
    구글 드라이브에서 폴더를 다운로드받는 것과 비슷해요. 하지만 Git은 단순 다운로드가 아니라 '역사'까지 함께 가져온다는 점이 달라요.

---

### 3. 프로젝트 폴더로 들어가기
이제 코드를 가져왔으니, 방금 생긴 `hra-website` 폴더 안으로 직접 들어가야 해요.

```bash title="폴더 이동"
cd hra-website
```

---

### 4. VS Code로 열기
이 폴더를 스마트 메모장인 VS Code에서 열어볼게요.

```bash title="VS Code 실행"
code .
```
!!! tip "안 되나요?"
    만약 `code` 명령어가 안 먹힌다면, VS Code를 직접 실행한 뒤 **File(파일) → Open Folder(폴더 열기)**를 눌러 바탕화면의 `hra-website` 폴더를 선택하세요.

---

### 5. 필요한 재료(npm) 설치하기
우리 웹사이트를 만들기 위한 밀가루, 설탕 같은 재료들을 한 번에 다운로드받는 과정이에요.

```bash title="패키지 설치"
npm install
```
!!! info "이게 뭔가요?"
    프로젝트 폴더 안에 `node_modules`라는 이름의 커다란 폴더가 생길 거예요. 이게 바로 우리 사이트가 돌아가게 해주는 수많은 외부 재료들이랍니다.

---

### 6. 환경 변수(.env) 설정
서버와 통신하기 위한 비밀 열쇠(키)를 설정하는 단계예요.

```bash title="설정 파일 복사"
cp .env.example .env
```
!!! warning "관리자에게 문의"
    `.env` 파일을 만들었다면 팀장님께 안에 채워야 할 내용을 물어보세요. 이 파일은 비밀이 담겨 있어서 GitHub에 올라가지 않거든요.

---

### 7. 에러가 나면?
- **'git' is not recognized**: Git 설치가 안 됐거나 터미널을 다시 켜야 해요.
- **Permission denied**: 관리자 권한이 없어서 그럴 수 있어요. `sudo`를 앞에 붙이거나, 바탕화면이 아닌 다른 폴더에서 시도해보세요.
- **npm install 실패**: Node.js 버전이 낮을 수 있어요. 최신 LTS 버전인지 다시 확인해보세요.
