# 사이트 실행하기 — 내 컴퓨터에서 사이트 띄우기

모든 준비가 끝났어요! 이제 우리 컴퓨터에서 직접 웹사이트를 실행하고, 내용을 수정해보는 시간이에요.

---

### 1. 개발 서버 실행하기
터미널에서 `hra-website` 폴더로 들어가 있는 상태에서 다음 명령어를 입력하세요.

```bash title="개발 서버 실행"
npm run dev
```
!!! success "이런 문구가 나오면 성공!"
    `ready - started server on 0.0.0.0:3000, url: http://localhost:3000` 같은 메시지가 나오면 우리 사이트가 실행된 거예요!

---

### 2. 브라우저에서 확인하기
크롬이나 사파리를 열고 주소창에 다음 주소를 입력하세요.

[http://localhost:3000](http://localhost:3000)

---

### 3. 실습: 직접 글자 바꿔보기
우리 사이트의 내용을 내 마음대로 바꿔보고, 실시간으로 바뀌는 걸 확인해볼까요?

1. VS Code에서 `src/app/(public)/page.tsx` 파일을 찾아서 열어요.
2. 한글로 된 텍스트(예: "안녕하세요", "HRA" 등)를 찾아서 "반가워요! 내 이름은 홍길동이에요"라고 바꿔보세요.
3. **저장(Ctrl + S 또는 Cmd + S)**을 누르세요.
4. 브라우저로 돌아가 보세요. 새로고침을 하지 않아도 글자가 바로 바뀌어 있을 거예요!

---

### 4. 서버 끄는 법
개발을 멈추고 싶을 때는 터미널 창을 클릭하고 **Ctrl + C**를 누르세요. 그러면 서버가 멈추고, 더 이상 `localhost:3000`으로 접속할 수 없게 돼요.

---

### 5. 다시 시작할 때
나중에 다시 개발을 하고 싶을 때는 이렇게만 하면 돼요.

1. 터미널을 열어요.
2. 바탕화면의 프로젝트 폴더로 가요: `cd Desktop/hra-website`
3. 개발 서버를 켜요: `npm run dev`

---

### 6. 자주 나는 에러 해결법
- **Port 3000 already in use**: 이미 다른 서버가 3000번 포트를 쓰고 있다는 뜻이에요. 터미널을 완전히 껐다가 다시 켜보거나, 열려 있는 다른 터미널 창을 찾아 끄세요.
- **Module not found**: 재료(npm)가 부족해서 그래요. 다시 한번 `npm install`을 입력해서 재료를 다 가져오세요.
- **Node.js 버전 오류**: Node.js가 너무 옛날 버전일 수 있어요. [nodejs.org](https://nodejs.org/)에서 다시 LTS 버전을 설치하세요.

!!! tip "그래도 안 된다면?"
    언제든지 팀장님이나 선배님께 에러 메시지를 캡처해서 물어보세요. 개발은 원래 에러와 함께하는 거니까요! 😊
