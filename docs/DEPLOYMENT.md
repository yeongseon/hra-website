# HRA 웹사이트 배포 및 문제 해결 가이드

이 문서는 개발을 처음 시작하는 대학생분들이 프로젝트를 인터넷에 실제로 올리고(배포), 발생하는 문제들을 스스로 해결할 수 있도록 돕기 위해 작성되었습니다.

---

## 1. Vercel 배포 완전 가이드

### Vercel이란?
Vercel은 우리가 만든 웹사이트 코드를 실제 인터넷 주소로 접속할 수 있게 만들어주는 '배포 플랫폼'입니다. GitHub에 코드를 올리면 Vercel이 이를 감지해 자동으로 최신 버전을 사이트에 반영해줍니다.

### 배포 순서

1.  **Vercel 회원가입 및 로그인**
    *   [Vercel 홈페이지](https://vercel.com/)에 접속합니다.
    *   **Continue with GitHub**를 선택해 본인의 GitHub 계정으로 가입하고 로그인합니다.

2.  **프로젝트 연결 (Import)**
    *   대시보드에서 오른쪽 상단의 **Add New...** 버튼을 누르고 **Project**를 선택합니다.
    *   **Import Git Repository** 목록에서 `hra-website` 저장소를 찾아 **Import** 버튼을 누릅니다.

3.  **환경변수(Environment Variables) 설정**
    *   배포 설정 화면 중간에 **Environment Variables** 섹션이 있습니다.
    *   여기에 아래 '2. 환경변수 상세 설명' 섹션에 있는 4가지 변수를 하나씩 추가해야 합니다.
    *   **Key**에는 변수 이름(예: `DATABASE_URL`), **Value**에는 해당 값을 넣고 **Add**를 누릅니다.
    *   체크박스(Production, Preview, Development)는 기본값인 전체 체크 상태로 둡니다.

4.  **배포 시작**
    *   설정이 끝났다면 하단의 **Deploy** 버튼을 누릅니다.
    *   약 2~3분 정도 기다리면 폭죽 애니메이션과 함께 배포가 완료됩니다!

5.  **자동 배포 확인**
    *   이제 본인의 컴퓨터에서 코드를 수정하고 GitHub에 `git push`를 하면, Vercel이 이를 자동으로 감지해 사이트를 다시 빌드하고 업데이트합니다.

## 2. 환경변수 상세 설명

환경변수는 데이터베이스 주소나 비밀번호처럼 겉으로 드러나면 안 되는 중요한 정보를 안전하게 보관하는 상자입니다.

### 1) DATABASE_URL (데이터베이스 주소)
*   **역할**: 웹사이트가 데이터를 저장하고 불러올 창고(Neon Postgres)의 주소입니다.
*   **가져오는 법**:
    1.  [Neon](https://neon.tech/)에 로그인합니다.
    2.  생성한 프로젝트의 대시보드 메인 화면을 봅니다.
    3.  **Connection String** 부분에서 `postgres://...`로 시작하는 긴 주소를 복사합니다. (비밀번호가 포함된 모드인지 확인하세요!)

### 2) AUTH_SECRET (보안 비밀키)
*   **역할**: 사용자의 로그인을 안전하게 유지하기 위한 암호화 키입니다.
*   **생성하는 법**:
    1.  내 컴퓨터의 터미널(VS Code 터미널)에서 아래 명령어를 입력합니다.
        ```bash
        npx auth secret
        ```
    2.  나오는 랜덤한 문자열을 복사해서 사용합니다.

### 3) BLOB_READ_WRITE_TOKEN (이미지 저장소 토큰)
*   **역할**: 갤러리나 공지사항에 이미지를 업로드할 때 사용하는 저장소(Vercel Blob)의 접근 권한입니다.
*   **이것이 없으면**: 갤러리에서 이미지 업로드 시 에러가 발생합니다. 텍스트 기반 기능(공지사항 글 작성, 회원가입 등)은 정상 동작합니다.
*   **생성 방법 (단계별)**:
    1. [Vercel 대시보드](https://vercel.com/)에 로그인합니다.
    2. 상단 메뉴에서 **Storage** 탭을 클릭합니다.
    3. **Create** 버튼을 누르고, 저장소 유형으로 **Blob**을 선택합니다.
    4. 이름을 입력합니다 (예: `hra-images`). 리전은 기본값 그대로 둡니다.
    5. **Create** 버튼을 눌러 생성을 완료합니다.
    6. 생성이 완료되면 화면에 `BLOB_READ_WRITE_TOKEN` 값이 표시됩니다. 이 값을 복사합니다.
    7. **프로젝트에 연결하기**:
       - Vercel 대시보드 → 프로젝트(hra-website) 선택
       - **Settings** → **Environment Variables** 클릭
       - Key: `BLOB_READ_WRITE_TOKEN`, Value: 복사한 토큰 값 붙여넣기
       - **Save** 클릭
    8. **재배포 필수**: 환경변수를 추가한 후에는 **Deployments** 탭에서 최신 배포의 점 세 개(...) → **Redeploy** 클릭
*   **확인 방법**: 관리자 페이지(`/admin/gallery`)에서 이미지를 업로드해보세요. 정상적으로 올라가면 설정 완료!

### 4) NEXT_PUBLIC_APP_URL (사이트 주소)
*   **역할**: 이메일 전송이나 링크 공유 시 기준이 되는 사이트 주소입니다.
*   **설정법**: 
    *   배포 전이라면 `https://hra-website-theta.vercel.app` (또는 본인의 Vercel 주소)를 넣습니다.
    *   로컬 개발 환경에서는 `http://localhost:3000`을 사용합니다.

## 3. 커스텀 도메인 연결 (선택 사항)

기본으로 제공되는 `.vercel.app` 주소 대신 `my-hra.com` 같은 주소를 쓰고 싶을 때 사용합니다.

1.  **도메인 구입**: 가비아, 후이즈 등에서 도메인을 먼저 구입해야 합니다.
2.  **Vercel 설정**: 프로젝트 대시보드 -> **Settings** -> **Domains**에서 구입한 도메인을 입력하고 **Add**를 누릅니다.
3.  **DNS 설정**: Vercel이 안내하는 `A 레코드`나 `CNAME` 값을 도메인 구입처의 관리 페이지(DNS 설정)에 입력합니다.
4.  **적용 대기**: 전 세계 인터넷망에 반영되기까지 짧게는 10분, 길게는 하루 정도 걸릴 수 있습니다.

## 4. Vercel 대시보드 활용하기

*   **배포 상태 확인**: **Deployments** 탭에서 현재 사이트가 정상적으로 빌드되었는지, 에러가 났는지 확인할 수 있습니다.
*   **로그(Logs) 확인**: 사이트가 안 열릴 때 **Logs** 메뉴를 보면 빨간색 에러 메시지가 떠 있습니다. 개발자들은 여기서 문제의 원인을 찾습니다.
*   **재배포(Redeploy)**: 가끔 알 수 없는 이유로 배포가 꼬였을 때, 특정 배포 항목 옆의 점 세 개(...) 버튼을 눌러 **Redeploy**를 할 수 있습니다.
*   **환경변수 수정**: **Settings** -> **Environment Variables**에서 언제든 값을 바꿀 수 있습니다. (값을 바꾼 후에는 반드시 다시 배포(Redeploy)해야 적용됩니다.)
*   **롤백(Rollback)**: 업데이트한 내용에 큰 버그가 있다면, 이전의 정상적이었던 버전 옆의 **Rollback** 버튼을 눌러 즉시 되돌릴 수 있습니다.

## 5. 트러블슈팅 (에러 해결법)

### ⚠️ "DATABASE_URL is not set" 또는 접속 에러
*   **원인**: 환경변수가 제대로 설정되지 않았거나, 데이터베이스 서버가 꺼져 있는 경우입니다.
*   **해결**: Vercel Settings에서 `DATABASE_URL`이 정확한지 확인하고, Neon 사이트에서 데이터베이스가 'Active' 상태인지 확인하세요.

### ⚠️ "relation [테이블명] does not exist"
*   **원인**: 코드는 수정했는데, 실제 데이터베이스에 표(Table)를 만들지 않았을 때 발생합니다.
*   **해결**: 내 컴퓨터 터미널에서 `npx drizzle-kit push`를 실행하여 데이터베이스 구조를 최신화하세요.

### ⚠️ "This page couldn't load" (500 에러)
*   **원인**: 서버 코드에서 예상치 못한 오류가 발생한 것입니다.
*   **해결**: Vercel 대시보드의 **Logs** 탭을 열어 어떤 파일의 몇 번째 줄에서 에러가 났는지 확인하세요.

### ⚠️ 로그인이나 회원가입이 안 됨
*   **원인**: `AUTH_SECRET`이 없거나 형식이 잘못된 경우입니다.
*   **해결**: `npx auth secret`으로 새로 만든 키를 Vercel 환경변수에 등록하고 재배포하세요.

### ⚠️ 이미지 업로드가 실패함
*   **원인**: `BLOB_READ_WRITE_TOKEN` 설정이 누락되었거나 만료되었습니다.
*   **해결**: Vercel Storage 탭에서 Blob 저장소가 프로젝트와 잘 연결되어 있는지 확인하세요.

### ⚠️ "Port 3000 already in use" (내 컴퓨터에서 실행 시)
*   **원인**: 이미 다른 터미널에서 웹사이트를 실행 중이거나, 이전 실행이 비정상적으로 종료되어 포트가 닫히지 않았습니다.
*   **해결**:
    *   **Windows**: 터미널에 `netstat -ano | findstr :3000` 입력 후 나오는 PID를 확인해 `taskkill /F /PID [번호]` 실행
    *   **macOS**: 터미널에 `lsof -i :3000` 입력 후 PID 확인해 `kill -9 [번호]` 실행

## 6. 로컬 개발 환경 vs 배포 환경

| 구분 | 로컬 개발 (Local) | 실제 배포 (Production) |
| :--- | :--- | :--- |
| **주소** | `localhost:3000` | `hra-website-theta.vercel.app` |
| **변수 저장** | `.env` 파일 | Vercel 대시보드 환경변수 설정 |
| **목적** | 기능 개발 및 테스트 | 실제 사용자에게 서비스 제공 |

**주의**: "내 컴퓨터(localhost)에서는 잘 되는데 배포하니까 안 돼요!" 하는 경우는 99% **환경변수 설정**을 빠뜨렸거나 오타가 난 경우입니다.

## 7. 보안 주의사항 (매우 중요)

1.  **`.env` 파일은 절대 공유하지 마세요**: 이 파일에는 여러분의 데이터베이스 비밀번호와 보안 키가 들어있습니다.
2.  **GitHub에 올리지 마세요**: `.gitignore` 파일에 `.env`가 등록되어 있는지 꼭 확인하세요. 만약 GitHub에 유출되었다면 즉시 비밀번호를 바꾸고 새로운 키를 발급받아야 합니다.
3.  **공공장소 주의**: 공용 컴퓨터에서 Vercel이나 GitHub 로그인을 했다면 반드시 로그아웃하세요.

## 8. 배포 후 첫 운영 체크리스트

사이트를 Vercel에 처음 배포한 후, 실제 운영을 시작하기 전에 아래 단계를 순서대로 완료하세요.

### 체크리스트

| 순서 | 할 일 | 완료 확인 |
| :---: | :--- | :--- |
| 1 | 데이터베이스 테이블 생성 (`npx drizzle-kit push`) | `relation does not exist` 에러 없음 |
| 2 | 사이트 접속 확인 (Vercel URL로 접속) | 홈페이지가 정상적으로 보임 |
| 3 | 회원가입 테스트 (`/register`) | 가입 완료 후 로그인 가능 |
| 4 | 첫 관리자 계정 지정 (Neon SQL Editor) | `/admin` 접속 가능 |
| 5 | Blob 저장소 생성 (Vercel Storage) | 갤러리 이미지 업로드 가능 |
| 6 | 기수 생성 (`/admin/recruitment` → 새 기수) | 모집안내 페이지에 표시됨 |
| 7 | 공지사항 작성 테스트 (`/admin/notices`) | 공지사항 페이지에 글이 보임 |

### 각 단계 상세 설명

**1단계: 데이터베이스 테이블 생성**
- 내 컴퓨터 터미널에서 프로젝트 폴더로 이동
- `.env` 파일에 `DATABASE_URL`이 설정되어 있는지 확인
- 아래 명령어 실행:
  ```bash
  npx drizzle-kit push
  ```
- "Changes applied" 메시지가 나오면 성공

**2단계: 사이트 접속 확인**
- Vercel이 알려준 URL (예: `https://hra-website-theta.vercel.app`)로 접속
- 홈페이지가 보이면 OK
- 500 에러가 나면 → Vercel Settings에서 환경변수가 모두 설정되어 있는지 확인

**3단계: 회원가입 테스트**
- `/register` 접속 → 이름, 이메일, 비밀번호 입력 → 회원가입
- `/login` 접속 → 방금 가입한 이메일/비밀번호로 로그인
- 정상 로그인되면 성공

**4단계: 첫 관리자 계정 지정**
- [neon.tech](https://neon.tech/) 접속 → SQL Editor 열기
- 아래 SQL 실행 (이메일은 3단계에서 가입한 것으로):
  ```sql
  UPDATE users SET role = 'ADMIN' WHERE email = '내이메일@example.com';
  ```
- 사이트에서 로그아웃 → 다시 로그인 → 상단에 "관리자" 메뉴가 보이면 성공
- 자세한 방법은 DATABASE.md의 "관리자 계정 직접 만들기" 참고

**5단계: Blob 저장소 생성**
- 이 문서의 섹션 2-3 "BLOB_READ_WRITE_TOKEN" 내용을 따라 설정
- 설정 후 `/admin/gallery`에서 이미지 업로드 테스트
- (이 단계를 건너뛰어도 텍스트 기능은 모두 정상 동작합니다)

**6단계: 기수 생성**
- `/admin/recruitment` → "새 기수 등록" 클릭
- 기수 이름, 기간, 모집 상태 입력 → 저장
- 메인 사이트의 `/recruitment` 페이지에서 확인

**7단계: 공지사항 작성 테스트**
- `/admin/notices` → "새 공지사항" 클릭
- 제목과 내용 입력 → 저장
- 메인 사이트의 `/notices` 페이지에서 확인

모든 단계를 완료하면 사이트가 운영 준비 완료된 것입니다!
