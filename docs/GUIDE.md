## 1. 코드를 안전하게 관리하기 (Git)

**Git**은 코드 버전을 관리하는 프로그램이에요. 워드에서 "변경 내용 추적" 기능 써본 적 있죠? 그거랑 비슷한 거예요. 뭔가 잘못 고쳐서 사이트가 망가져도 과거 버전으로 되돌릴 수 있어요.

### ⭐ 꼭 알아야 할 명령어 3가지

아래 명령어들은 **터미널**(컴퓨터에 글자로 명령을 내리는 검은 창)에 입력하는 거예요.

**① 코드를 내 컴퓨터로 복사하기 (처음 한 번만)**

```bash
git clone https://github.com/사용자명/hra-website.git
```

> 👉 이 명령어를 치면 내 컴퓨터에 `hra-website` 폴더가 생기고, 그 안에 사이트 코드가 다운받아져요.

**② 내가 수정한 내용 저장하기**

```bash
git add .
git commit -m "메인 페이지 로고 수정"
```

> 👉 `git add .`은 "내가 바꾼 파일들을 모아둬"라는 뜻이고, `git commit`은 "이렇게 바꿨다"라고 기록을 남기는 거예요. 따옴표 안에 뭘 바꿨는지 메모를 적으면 나중에 찾기 편해요.

**③ 내 수정사항을 팀 저장소에 올리기**

```bash
git push origin main
```

> 👉 내 컴퓨터에서 작업한 걸 인터넷에 있는 팀 저장소(GitHub)에 올리는 거예요. 올리면 Vercel이 자동으로 사이트를 최신 버전으로 바꿔줘요!

### 브랜치(복사본 작업) — 나중에 알면 되는 것

브랜치는 원본을 건드리지 않고 따로 작업하는 복사본이에요. 시험 전에 연습장에 먼저 풀어보는 것과 같아요.

```bash
git checkout -b 새기능이름
```

> 👉 이 명령어를 치면 원본(`main`)은 그대로 두고, "새기능이름"이라는 복사본에서 안전하게 작업할 수 있어요.

### 충돌이 나면?

두 사람이 같은 파일의 같은 줄을 동시에 고치면 "충돌"이 생겨요. 무서운 게 아니에요!

1. VS Code를 열면 충돌 부분이 색깔로 표시돼요
2. 어느 쪽 코드를 쓸지 골라서 저장하면 돼요
3. 그 다음 `git add .` → `git commit -m "충돌 해결"` 하면 끝!

> 💡 모르겠으면 팀원한테 물어보세요. 충돌은 흔한 일이라서 부끄러워할 필요 없어요.

## 2. ⭐ 개발 환경 세팅하기 (처음 한 번만)

### 먼저 이 2가지를 설치하세요

| 프로그램 | 뭐 하는 건가요? | 다운로드 |
| --- | --- | --- |
| **Node.js** | 사이트를 돌리는 엔진이에요. 자동차의 엔진 같은 거예요. **LTS 버전**을 설치하세요. | [nodejs.org](https://nodejs.org/ko/) |
| **VS Code** | 코드를 보고 수정하는 프로그램이에요. 메모장의 고급 버전이라고 생각하면 돼요. | [code.visualstudio.com](https://code.visualstudio.com/) |

### VS Code에 추가하면 좋은 것들 (선택사항)

VS Code 왼쪽에 네모 4개짜리 아이콘이 있어요. 그걸 누르면 검색창이 나오는데, 아래 이름을 검색해서 **Install** 버튼을 누르세요.

- **Korean Language Pack** — VS Code 메뉴가 한국어로 바뀌어요
- **Tailwind CSS IntelliSense** — 디자인 코드를 자동으로 완성해줘요
- **ESLint** — 문법 실수가 있으면 빨간 밑줄로 알려줘요

### 터미널이 뭔가요?

터미널은 마우스 대신 **글자로 컴퓨터에 명령을 내리는 창**이에요. 카카오톡 채팅처럼 명령어를 입력하면 컴퓨터가 대답해줘요.

- **Mac** → Spotlight(Cmd + Space)에서 "터미널" 검색
- **Windows** → 시작 메뉴에서 "PowerShell" 검색

자주 쓰는 터미널 명령어:

| 명령어 | 뭐 하는 건가요? |
| --- | --- |
| `ls` (Mac) / `dir` (Windows) | 지금 폴더에 뭐가 있는지 보기 |
| `cd 폴더이름` | 그 폴더 안으로 들어가기 |

### ⭐ 사이트 실행하기 (3단계)

VS Code에서 터미널을 열어요 (상단 메뉴 → 터미널 → 새 터미널, 또는 Ctrl + `).

```bash
# 1단계: 필요한 재료를 자동으로 다운받기 (처음 한 번만)
npm install
```

> 👉 요리하려면 재료가 필요하죠? `npm install`은 사이트를 만드는 데 필요한 재료들을 인터넷에서 자동으로 가져와요.

```bash
# 2단계: 설정 파일 만들기 (처음 한 번만)
cp .env.example .env
```

> 👉 `.env` 파일은 사이트가 작동하려면 필요한 비밀 설정값(비밀번호 모음집)이에요. 안의 내용은 프로젝트 관리자한테 물어보세요!

```bash
# 3단계: 내 컴퓨터에서 사이트 켜기
npm run dev
```

> 👉 이 명령어를 치면 내 컴퓨터에서 사이트가 켜져요.

**브라우저**(크롬, 사파리 등)를 열고 주소창에 `http://localhost:3000` 을 입력하면 사이트가 보여요! 🎉

## 3. 폴더 구조 한눈에 보기

프로젝트 폴더 안에 뭐가 들어있는지 대충 알아둬요.

```
src/
├── app/                  ← 📄 페이지들이 들어있는 곳
│   ├── (public)/         ← 누구나: 홈, 소개, 교수진, 커리큘럼, 기수, 모집, 공지, 갤러리
│   ├── (auth)/           ← 로그인 페이지
│   ├── (admin)/          ← 관리자 전용
│   └── (member)/         ← 로그인 필요: 자료실, 수업일지, 마이페이지
├── features/             ← ⚙️ 기능 코드 (지원서 제출, 회원 관리 등)
├── lib/                  ← 🔧 설정 파일들 (DB 연결, 로그인 설정 등)
├── components/           ← 🧩 재사용되는 부품 (헤더, 푸터, 버튼 등)
└── scripts/              ← 📜 프로젝트 관리 스크립트 (맞춤법 검사 등)
```

> 💡 폴더 이름에 `(괄호)`가 있는 건 뭔가요?
> → 그냥 정리용이에요! 주소창에는 나타나지 않아요. 예를 들어 `(public)/about/page.tsx` 파일은 `사이트주소/about` 으로 접속하면 보여요.

## 4. ⭐ 페이지 수정하기

### 글자 바꾸기

홈페이지 문구를 바꾸고 싶으면:

1. VS Code에서 `src/app/(public)/page.tsx` 파일을 열어요
2. 한글 텍스트를 찾아서 원하는 내용으로 바꿔요
3. `Ctrl + S`(Mac은 Cmd + S)를 눌러 저장해요

> 👉 저장하면 브라우저에서 바로 바뀐 걸 볼 수 있어요! (사이트가 켜져 있을 때)

### 상단 메뉴 바꾸기

상단 메뉴는 `src/components/layout/header.tsx` 파일에 있어요.

```tsx
const navLinks = [
  { href: "/about", label: "소개" },
  { href: "/faculty", label: "교수진" },
  { href: "/curriculum", label: "커리큘럼" },
  { href: "/cohorts", label: "기수 소개" },
  { href: "/recruitment", label: "모집안내" },
  { href: "/notices", label: "공지사항" },
  { href: "/gallery", label: "갤러리" },
  { href: "/resources", label: "자료실" },
];
```

> 👉 `href`는 주소, `label`은 메뉴에 표시되는 이름이에요. 새 메뉴를 추가하고 싶으면 위와 같은 형식으로 한 줄 추가하면 돼요.

> 💡 **맞춤법 검사**: 글을 다 썼다면 터미널에 `npm run spell-check`를 입력해 보세요! 틀린 맞춤법이 있는지 자동으로 찾아줘요. `npm run spell-fix`를 치면 자동으로 고쳐주기까지 한답니다! ✨

> 💡 로그인/로그아웃 버튼, 수업일지 메뉴, 관리자 메뉴는 자동으로 표시돼요. 따로 설정할 필요 없어요!

### 하단(푸터) 수정

`src/components/layout/footer.tsx`에서 이메일, SNS 링크 등을 바꿀 수 있어요.

### 새 페이지 만들기

`/hello` 라는 새 페이지를 만들고 싶으면:

1. `src/app/(public)/hello/` 폴더를 만들어요
2. 그 안에 `page.tsx` 파일을 만들고 아래 내용을 붙여넣어요

```tsx
export default function HelloPage() {
  return (
    <div className="p-10 text-center">
      <h1 className="text-4xl font-bold">안녕하세요!</h1>
      <p>새로 만든 페이지입니다.</p>
    </div>
  );
}
```

> 👉 저장하고 브라우저에서 `localhost:3000/hello`로 접속하면 새 페이지가 보여요!

## 5. 디자인 바꾸기

이 사이트는 **디자인 도구**(Tailwind CSS)를 사용해요. 코드 안에서 `className="..."`이라고 적힌 부분이 디자인을 담당해요.

### 자주 쓰는 디자인 코드

| 하고 싶은 것 | 코드 | 설명 |
| --- | --- | --- |
| 글자를 파란색으로 | `text-blue-500` | 숫자가 클수록 진해요 |
| 글자를 크게 | `text-2xl` | sm, lg, xl, 2xl... 순서대로 커져요 |
| 배경을 검정으로 | `bg-black` | |
| 안쪽 여백 주기 | `p-4` | 숫자가 클수록 여백이 넓어요 |
| 위쪽만 여백 | `mt-10` | mt = margin-top (위쪽 바깥 여백) |
| 요소를 가로로 나열 | `flex` | 세로로 쌓이던 걸 가로로 배치 |
| 모서리 둥글게 | `rounded-lg` | |

### 화면 크기별로 다르게 보이게 하기

`sm:`, `md:`, `lg:` 같은 접두어를 붙이면 화면 크기에 따라 다르게 보여요.

```tsx
className="text-sm md:text-lg"
```

> 👉 핸드폰 같은 작은 화면에서는 `text-sm`(작은 글자), PC 같은 큰 화면에서는 `text-lg`(큰 글자)로 보여요.

## 6. ⭐ 관리자 페이지 사용법

### 관리자가 되려면 (처음 한 번만)

**1단계:** 사이트에서 구글 또는 카카오로 로그인해요.

> 👉 처음 로그인하면 자동으로 일반 회원 계정이 만들어져요.

**2단계:** [neon.tech](https://neon.tech/)에 로그인해요.

1. 왼쪽 메뉴에서 프로젝트를 선택해요
2. **SQL Editor** 를 클릭해요
3. 아래 명령어를 복사해서 붙여넣고 **Run** 버튼을 눌러요

```sql
UPDATE users SET role = 'ADMIN' WHERE email = '내이메일@example.com';
```

> 👉 `내이메일@example.com` 부분을 본인이 로그인할 때 쓴 이메일로 바꿔요!

**3단계:** 사이트에서 로그아웃했다가 다시 로그인하면 상단에 **"관리자"** 링크가 생겨요!

### 관리자 메뉴에서 할 수 있는 것들

| 메뉴 | 뭘 할 수 있나요? |
| --- | --- |
| **대시보드** | 사이트 전체 현황을 한눈에 봐요 |
| **공지사항** | 글을 쓰고, 수정하고, 삭제할 수 있어요 |
| **수업일지** | 수업 내용을 기록해요 |
| **기수 관리** | 새 기수를 만들고, 모집 상태를 바꿔요 |
| **갤러리** | 활동 사진을 올려요 |
| **지원서** | 접수된 지원서를 보고, 합격/불합격 처리를 해요 |
| **회원 관리** | 전체 회원 목록을 보고, 역할(관리자/멤버)을 바꿔요 |
| **개발 문서** | 이 가이드를 사이트 안에서 바로 읽어요 |

### 자주 묻는 질문

**Q: 로그인했는데 관리자 메뉴가 안 보여요.**
→ 로그아웃했다가 다시 로그인해보세요. 로그인할 때 권한 정보가 새로 받아져요.

**Q: 다른 사람도 관리자로 만들 수 있나요?**
→ 네! 두 가지 방법이 있어요:
1. **쉬운 방법**: 관리자 페이지 → 회원 관리 → 원하는 사람 옆의 "관리자로 변경" 버튼 클릭!
2. **처음 한 번은**: 위의 SQL 명령어로 직접 지정해야 해요 (아직 관리자가 아무도 없을 때)

> ⚠️ 단, 그 사람이 먼저 한 번은 사이트에 로그인해서 계정이 만들어져 있어야 해요!

**Q: 관리자 권한을 없애려면?**
→ 관리자 페이지 → 회원 관리 → "멤버로 변경" 버튼을 누르면 돼요.

**Q: 지원서 합격/불합격은 어떻게 하나요?**
→ **현재 모집 안내 페이지의 '지원하기' 버튼은 구글 폼으로 연결되어 있어요!** 접수된 내용은 구글 폼 응답 시트에서 확인하시면 돼요. (사이트 내 지원서 관리 기능은 추후 업데이트 예정이에요)

## 7. 이런 메시지가 뜨면 이렇게 하세요

### "Module not found" (모듈을 찾을 수 없습니다)

> 👉 터미널에 `npm install`을 다시 입력해보세요. 필요한 재료가 빠졌을 수 있어요.

### "Port 3000 is already in use" (포트가 이미 사용 중)

> 👉 이전에 켠 사이트가 아직 돌고 있어서 그래요. 터미널에서 `Ctrl + C`를 눌러서 끄고, `npm run dev`를 다시 입력해보세요.

### 수정했는데 화면이 안 바뀌어요

> 👉 파일을 저장(`Ctrl + S`)했는지 확인해보세요! 그래도 안 되면 브라우저에서 `Ctrl + Shift + R`을 눌러서 강력 새로고침해보세요.

### 비밀 설정값 관련 에러

> 👉 `.env` 파일이 있는지, 안에 필요한 값들이 다 채워져 있는지 확인하세요. 잘 모르겠으면 프로젝트 관리자한테 물어보세요!
