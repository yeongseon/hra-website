# HRA 웹사이트 개발 가이드 (초보자용)

안녕하세요! 이 가이드는 **개발을 처음 시작하는 대학생** 여러분이 HRA(Human Renaissance Academy) 웹사이트 프로젝트를 직접 수정하고 관리할 수 있도록 돕기 위해 작성되었습니다. 복잡한 이론보다는 **"무엇을 설치하고, 어디를 수정해야 하는지"** 실무 위주로 아주 상세하게 설명합니다.

## 1. Git 사용법 (코드 관리의 시작)

### Git이란 무엇인가요?
Git은 코드의 **"타임머신"**이자 **"복사본 관리자"**입니다. 코드를 수정하다가 실수했을 때 과거로 되돌리거나, 여러 사람이 동시에 같은 프로젝트를 작업할 때 코드가 뒤섞이지 않게 도와줍니다.

### 주요 명령어 단계별 따라하기

1.  **코드 내려받기 (Clone)**
    GitHub에 있는 프로젝트를 내 컴퓨터로 가져오는 첫 단계입니다. 터미널(검은 창)을 열고 아래 명령어를 입력하세요.
    ```bash
    git clone https://github.com/사용자명/hra-website.git
    # 결과: 내 컴퓨터에 'hra-website' 폴더가 생기고 코드가 다운로드됩니다.
    ```

2.  **수정한 내용 기록하기 (Add & Commit)**
    파일을 수정했다면, Git에게 "이 변경사항을 기억해줘!"라고 말해야 합니다.
    ```bash
    git add .
    # 설명: 현재 폴더의 모든 변경사항을 기록 대기소에 올립니다.

    git commit -m "메인 페이지 로고 수정"
    # 설명: -m 뒤에 어떤 작업을 했는지 짧은 메모를 남깁니다. 나중에 이 메모를 보고 과거로 돌아갈 수 있습니다.
    ```

3.  **인터넷(GitHub)에 올리기 (Push)**
    내 컴퓨터에서 작업한 내용을 온라인 저장소에 반영합니다.
    ```bash
    git push origin main
    # 결과: GitHub 사이트에 접속했을 때 내가 수정한 코드가 보입니다.
    ```

### 브랜치(Branch)는 왜 쓰나요?
브랜치는 **"평행 우주"**와 같습니다. 원본 코드(`main`)를 건드리지 않고, 새로운 기능을 시험해볼 때 브랜치를 만들어 작업합니다. 작업이 잘 끝나면 나중에 `main`과 합칩니다.

### 충돌(Conflict)이 발생하면?
두 사람이 같은 파일의 같은 줄을 동시에 수정하고 올리면 발생합니다. 이때는 당황하지 말고 VS Code를 여세요. 충돌이 난 부분이 빨간색/파란색으로 표시됩니다. 둘 중 어떤 코드를 남길지 선택하고 다시 `add`와 `commit`을 하면 해결됩니다.

## 2. 개발 환경 설정

### 1단계: 필수 프로그램 설치
1.  **Node.js**: 웹사이트를 실행하는 엔진입니다. [Node.js 공식 사이트](https://nodejs.org/ko/)에서 **LTS 버전**을 다운로드하여 설치하세요.
2.  **VS Code (Visual Studio Code)**: 코드를 편집하는 메모장 같은 도구입니다. [다운로드 링크](https://code.visualstudio.com/)
3.  **VS Code 추천 확장 프로그램**: 왼쪽 메뉴의 사각형 아이콘(Extensions)을 누르고 아래 항목을 설치하면 코딩이 훨씬 쉬워집니다.
    *   **Korean Language Pack**: 메뉴를 한국어로 바꿉니다.
    *   **Tailwind CSS IntelliSense**: 디자인 코드를 자동 완성해줍니다.
    *   **ESLint**: 코드의 문법 오류를 찾아줍니다.

### 2단계: 터미널 기초 명령어
터미널은 마우스 대신 글자로 컴퓨터에 명령을 내리는 창입니다. (Mac: Terminal, Windows: CMD 또는 PowerShell)
*   `ls` (Windows는 `dir`): 현재 폴더에 어떤 파일이 있는지 보여줍니다.
*   `cd 폴더이름`: 해당 폴더 안으로 들어갑니다. (예: `cd hra-website`)
*   `mkdir 폴더이름`: 새 폴더를 만듭니다.

### 3단계: 프로젝트 실행하기
1.  VS Code에서 `hra-website` 폴더를 엽니다.
2.  상단 메뉴의 **터미널 > 새 터미널**을 누릅니다.
3.  아래 명령어를 순서대로 입력하세요.
    ```bash
    npm install
    # 설명: 프로젝트에 필요한 도구들을 한꺼번에 설치합니다. (처음 한 번만 실행)

    cp .env.example .env
    # 설명: 설정 파일 복사본을 만듭니다. (내용은 운영진에게 문의하세요!)

    npm run dev
    # 설명: 개발 서버를 켭니다.
    ```
4.  브라우저 주소창에 `http://localhost:3000`을 입력하면 내가 만든 사이트가 보입니다!

## 3. 프로젝트 폴더 구조 완전 해설

프로젝트의 폴더 구조를 알면 어디를 수정해야 할지 바로 알 수 있습니다.

*   **`src/app/` (페이지 주소)**: 웹사이트의 메뉴(URL)를 결정합니다.
    *   `(public)/`: 누구나 볼 수 있는 페이지 (홈, 소개, 공지사항 등)
    *   `(auth)/`: 로그인, 회원가입 관련
    *   `(admin)/`: 관리자 전용 페이지 (`/admin`)
    *   `(member)/`: 로그인한 학생들만 보는 페이지
*   **`src/components/` (디자인 부품)**: 여러 페이지에서 공통으로 쓰는 부품들입니다.
    *   `layout/`: 헤더(메뉴바), 푸터(하단 정보)가 들어있습니다.
    *   `ui/`: 버튼, 입력창, 팝업창 등 기본 디자인 부품들입니다.
*   **`src/features/` (핵심 기능)**: 데이터베이스와 대화하는 코드가 들어있습니다.
    *   지원서 제출, 공지사항 등록 같은 실제 "동작"을 담당합니다.
*   **`src/lib/` (설정 도구)**: 데이터베이스 연결 설정이나 로그인 보안 설정이 들어있습니다.
*   **`public/` (파일 창고)**: 이미지, 아이콘, 폰트 같은 파일들을 넣어두는 곳입니다.

## 4. 페이지 수정하는 법 (실전 예시)

### 텍스트나 제목 바꾸기
예를 들어, 홈페이지의 메인 문구를 바꾸고 싶다면 `src/app/(public)/page.tsx` 파일을 엽니다. 그 안에서 한글로 적힌 텍스트를 찾아 원하는 내용으로 수정하고 저장(`Ctrl + S`)하세요. 브라우저를 새로고침하지 않아도 바로 바뀝니다!

### 네비게이션 메뉴 수정 (상단 메뉴)
메뉴 이름을 바꾸거나 새 메뉴를 추가하고 싶다면 `src/components/layout/header.tsx` 파일을 수정하세요.
*   왜 여기서 하나요? 헤더는 모든 페이지에 공통으로 나타나기 때문입니다.

### 하단 푸터(연락처, 주소) 수정
사이트 제일 아래에 있는 연락처나 저작권 정보를 바꾸려면 `src/components/layout/footer.tsx`를 수정합니다.

### 새 페이지 만들기
만약 `/hello`라는 주소의 페이지를 만들고 싶다면:
1.  `src/app/(public)/hello/` 폴더를 만듭니다.
2.  그 안에 `page.tsx` 파일을 만들고 아래 코드를 복사해 넣으세요.
    ```tsx
    export default function HelloPage() {
      return (
        <div className="p-10 text-center">
          <h1 className="text-4xl font-bold">안녕하세요!</h1>
          <p>새로 만든 페이지입니다.</p>
        </div>
      );
    }
    ```

## 5. 디자인/스타일 변경법 (Tailwind CSS)

이 프로젝트는 **Tailwind CSS**를 사용합니다. HTML 태그 안에 `className="..."`을 적어 디자인을 입히는 방식입니다.

### 자주 쓰는 디자인 코드 20개
*   **글자 색**: `text-blue-500` (파란색), `text-red-600` (빨간색)
*   **글자 크기**: `text-sm` (작게), `text-lg` (크게), `text-2xl` (매우 크게)
*   **배경 색**: `bg-gray-100` (연회색 배경), `bg-black` (검정 배경)
*   **간격**: `p-4` (안쪽 여백), `m-2` (바깥 여백), `mt-10` (위쪽만 바깥 여백)
*   **정렬**: `flex` (옆으로 나열), `items-center` (세로 가운데 정렬), `justify-between` (양 끝으로 벌리기)
*   **테두리**: `border` (선 긋기), `rounded-lg` (모서리 둥글게)

### 반응형 디자인 (모바일 대응)
`md:`라는 글자를 앞에 붙이면 "화면이 넓을 때(PC)만 적용해라"라는 뜻이 됩니다.
*   `className="text-sm md:text-lg"`: 모바일에서는 작은 글씨, PC에서는 큰 글씨로 보입니다.

## 6. 관리자 페이지 사용법 (`/admin`)

관리자 기능을 사용하려면 특별한 권한이 필요합니다. 개발을 전혀 모르더라도 아래 단계를 차근차근 따라하면 관리자 계정을 만들고 사이트를 관리할 수 있습니다.

### 관리자 계정 만들기 (최초 1회)

**1단계: 사이트에서 회원가입하기**
*   브라우저 주소창에 사이트 주소 뒤에 `/register`를 붙여 접속합니다. (예: `https://hra-website-theta.vercel.app/register`)
*   이름, 이메일, 비밀번호를 입력하고 "회원가입" 버튼을 클릭합니다.
*   가입 직후에는 **일반 회원(MEMBER)** 권한만 가진 상태입니다.

**2단계: Neon 대시보드에서 관리자 권한 부여**
*   [neon.tech](https://neon.tech/) 사이트에 접속하여 로그인합니다.
*   내 프로젝트를 선택한 뒤, 왼쪽 메뉴에서 **SQL Editor**를 클릭합니다.
*   입력창에 아래 SQL 명령어를 복사해서 넣고 **Run** 버튼을 클릭합니다. (이메일 주소는 본인이 가입한 것으로 수정하세요.)
    ```sql
    UPDATE users SET role = 'ADMIN' WHERE email = '내가가입한이메일@example.com';
    ```
*   하단에 "1 row affected"라는 메시지가 뜨면 성공입니다.
*   **확인 방법**: `SELECT email, role FROM users;` 명령어를 실행하여 내 이메일 옆의 role이 ADMIN으로 바뀌었는지 확인합니다.

**3단계: 관리자로 로그인**
*   사이트 주소 뒤에 `/login`을 붙여 접속합니다.
*   방금 권한을 변경한 이메일과 비밀번호로 로그인합니다.
*   상단 메뉴 바에 "관리자" 링크가 나타나면 모든 준비가 끝났습니다!

### 관리자 페이지 메뉴 설명

관리자 페이지에 접속하면 왼쪽 또는 상단 메뉴에서 아래 기능들을 사용할 수 있습니다.
*   **대시보드**: 전체 방문자 수나 최근 활동 등 사이트의 전반적인 현황을 요약해서 보여줍니다.
*   **공지사항**: 중요한 소식을 작성하고, 이미 올린 글을 수정하거나 삭제합니다.
*   **수업일지**: 기수별로 진행된 수업 내용을 기록하고 관리합니다.
*   **기수 관리**: 새로운 기수를 생성하거나, 현재 신입 회원을 모집 중인지 상태를 변경합니다.
*   **갤러리**: 활동 사진을 업로드하고 앨범 형태로 관리합니다.
*   **지원서**: 사이트를 통해 접수된 지원자들의 목록과 상세 내용을 확인합니다.
*   **개발 문서**: 지금 읽고 있는 이 가이드를 사이트 내부에서도 편하게 볼 수 있습니다.

### 자주 묻는 질문

*   **Q: 로그인했는데 관리자 메뉴가 안 보여요.**
    *   A: 권한(role)이 아직 ADMIN으로 변경되지 않은 것입니다. Neon SQL Editor에서 위 2단계를 다시 정확히 실행했는지 확인하세요.
*   **Q: 다른 사람도 관리자로 만들 수 있나요?**
    *   A: 네, 동일한 SQL 명령어에서 이메일 부분만 해당 사용자의 이메일로 바꿔서 실행하면 됩니다.
*   **Q: 관리자 권한을 해제하려면 어떻게 하나요?**
    *   A: 아래 명령어를 실행하면 다시 일반 회원으로 돌아갑니다.
    ```sql
    UPDATE users SET role = 'MEMBER' WHERE email = '해제할이메일@example.com';
    ```

## 7. 자주 하는 실수와 해결법

### "모듈을 찾을 수 없습니다 (Module not found)"
*   **원인**: 필요한 도구가 설치되지 않았거나 파일 경로가 틀렸을 때 발생합니다.
*   **해결**: 터미널에 `npm install`을 다시 입력해보세요.

### "포트 3000이 이미 사용 중입니다 (Port 3000 is already in use)"
*   **원인**: 이전에 켰던 개발 서버가 아직 종료되지 않았을 때 생깁니다.
*   **해결**: 터미널 창을 모두 닫거나 컴퓨터를 재부팅하세요. 혹은 터미널에서 `Ctrl + C`를 눌러 서버를 끄세요.

### 수정했는데 화면이 안 바뀌어요
*   **해결**: 파일을 저장(`Ctrl + S`)했는지 확인하세요. 간혹 브라우저 캐시 문제일 수 있으니 `Ctrl + Shift + R`로 강력 새로고침을 해보세요.

### 환경변수 에러 (Environmental variable missing)
*   **원인**: `.env` 파일에 필요한 정보(DB 주소 등)가 비어있을 때 발생합니다.
*   **해결**: `.env` 파일이 존재하는지, 그리고 필요한 항목들이 모두 채워져 있는지 확인하세요.
