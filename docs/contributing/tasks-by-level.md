# 난이도별 작업 가이드

HRA 웹사이트 프로젝트에 기여하는 방법은 다양해요. 본인의 숙련도에 맞춰 도전해볼 수 있는 작업들을 정리했습니다.

---

## ⭐ 1단계: 쉬움 (코딩 경험 없어도 가능)

웹 개발이 처음이라도 바로 시작할 수 있어요. 텍스트를 고치거나 새로운 소식을 올리는 것만으로도 큰 도움이 됩니다.

### 주요 작업
*   **오타 및 문구 수정**: 사이트 내 오타나 어색한 표현 수정
    *   경로: `src/app/(public)/` 폴더 내 페이지 파일
*   **공지사항 작성**: `content/notices/` 폴더에 Markdown(.md) 파일 추가
*   **이미지 교체**: `public/images/` 폴더 내 로고나 배너 이미지 업데이트
*   **버그 제보**: 발견한 문제를 GitHub Issue로 등록

### 💡 도구 활용
*   **OpenCode/Codex**: "이 문장을 더 자연스러운 한국어로 고쳐줘"라고 요청해보세요.
*   **Claude**: 작성한 공지사항의 맞춤법 검사를 부탁하면 편리합니다.

---

## ⭐⭐ 2단계: 보통 (HTML/CSS 기초 지식 필요)

태그 구조를 이해하고 스타일을 다룰 줄 안다면 도전해보세요. 기존 코드를 참고해서 수정하는 것부터 시작하면 좋습니다.

### 주요 작업
*   **스타일 수정**: Tailwind CSS를 사용해 간격이나 색상 조절
    *   대상: 컴포넌트의 `className` 속성
*   **페이지 추가**: 기존 페이지를 복사해 새로운 내용 구성
    *   경로: `src/app/(public)/` 폴더에 새 폴더와 `page.tsx` 생성
*   **컴포넌트 개선**: 버튼이나 입력창 등 공통 UI 요소 수정
    *   경로: `src/components/ui/` 폴더
*   **문서 보강**: `docs/` 폴더 내 가이드 문서 추가 및 수정

### 💡 도구 활용
*   **OpenCode**: "이 버튼을 더 둥글게 만드는 Tailwind 클래스를 알려줘"처럼 구체적인 스타일 수정을 물어보세요.
*   **Codex**: "기존 공지사항 페이지 코드를 참고해서 행사 안내 페이지 구조를 잡아줘"라고 요청해보세요.

---

## ⭐⭐⭐ 3단계: 어려움 (프로그래밍 경험 필요)

Next.js와 데이터베이스를 다뤄본 분들을 위한 작업입니다. 핵심 기능을 개발하거나 관리 시스템을 개선합니다.

### 주요 작업
*   **신규 기능 개발**: 데이터 모델 설계 및 핵심 로직 구현
    *   경로: `src/lib/db/schema.ts` (DB), `src/features/` (로직)
*   **API 엔드포인트 추가**: 서버 통신 로직 작성
    *   경로: `src/app/api/` 또는 `src/features/` 내 서버 액션
*   **관리자 기능 고도화**: 운영진용 관리 도구 기능 추가
    *   경로: `src/app/(admin)/` 폴더
*   **테스트 작성**: 기능 검증을 위한 테스트 코드 작성

### 💡 도구 활용
*   **Claude/Codex**: "Drizzle ORM 스키마에 새로운 테이블을 추가하는 코드를 작성해줘" 또는 "이 함수의 보안 취약점을 점검해줘"와 같은 기술적인 도움을 받아보세요.

---

작업 중 궁금한 점은 언제든 팀원들에게 물어봐주세요. 함께 더 좋은 사이트를 만들어가요!
