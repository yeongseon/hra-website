# 협업 규칙 — 브랜치 만들고 PR 올리기

함께 웹사이트를 만들어가는 과정에서 지켜야 할 약속들을 정리했어요. Git이나 GitHub이 처음이라도 괜찮아요. 아래 순서대로 차근차근 따라 해보세요.

---

### 1. 시작하기 전에: `main` 브랜치는 보호해야 해요
프로젝트의 중심인 `main` 브랜치는 실제 서비스에 바로 반영되는 코드가 모이는 곳이에요. 그래서 `main`에서 직접 코드를 수정하는 것은 피해야 합니다.
항상 **나만의 작업 브랜치**를 새로 만들어서 작업해 주세요.

---

### 2. 작업 브랜치 만들기
작업을 시작하기 전, 터미널에서 아래 명령어를 입력해 새 브랜치를 만듭니다.

```bash title="새 브랜치 만들기"
git checkout -b 브랜치이름
```

**브랜치 이름 규칙**
누구나 어떤 작업을 하는지 알 수 있게 지어주세요.
- **새 기능을 만들 때**: `feature/기능이름` (예: `feature/add-notice`)
- **버그를 고칠 때**: `fix/수정내용` (예: `fix/typo-homepage`)
- **디자인을 수정할 때**: `design/수정내용` (예: `design/header-color`)

---

### 3. 코드 수정하고 기록하기 (Add & Commit)
코드를 수정했다면 변경된 내용을 Git에 기록해야 해요.

1. **변경한 파일 선택하기**: 수정한 파일들을 저장 바구니에 담는 과정이에요.
    ```bash
    git add .
    ```
    (마침표 `.`는 모든 변경 내용을 포함한다는 뜻이에요.)

2. **메모 남기기 (Commit)**: 어떤 작업을 했는지 짧은 설명과 함께 저장합니다.
    ```bash
    git commit -m "홈페이지 오타 수정"
    ```

---

### 4. GitHub에 내 작업 올리기 (Push)
내 컴퓨터에 저장한 내용을 온라인 저장소(GitHub)로 보내야 다른 사람들도 볼 수 있어요.

```bash title="GitHub에 올리기"
git push origin 브랜치이름
```
(예: `git push origin feature/add-notice`)

---

### 5. Pull Request(PR) 만들기
GitHub 사이트에서 내가 만든 브랜치를 `main` 브랜치에 합쳐달라고 요청하는 단계예요.

1. [GitHub 저장소](https://github.com/yeongseon/hra-website)에 접속하세요.
2. 상단에 노란색 알림과 함께 **Compare & pull request** 버튼이 나타나면 클릭합니다.
3. **제목**: 작업 내용을 한눈에 알 수 있게 적어주세요.
4. **내용**: 수정한 부분이나 테스트 결과를 간단히 적어주세요.
5. 오른쪽 아래 **Create pull request** 버튼을 누르면 완료됩니다.

---

### 6. 코드 리뷰 받고 수정하기
팀원들이 코드를 보고 의견을 남길 수 있어요.
- 수정 제안이 들어오면 내 컴퓨터에서 코드를 고친 뒤, **다시 3번(Add & Commit)과 4번(Push)** 과정을 반복해 주세요.
- PR 페이지에 수정 내용이 자동으로 업데이트됩니다.

---

### 7. 마무리: 브랜치 삭제
모든 검토가 끝나고 코드가 `main`에 합쳐졌다면(Merged), 작업했던 브랜치는 삭제해도 좋습니다.
1. GitHub 사이트에서 **Delete branch** 버튼을 누르거나,
2. 내 컴퓨터 터미널에서 `main`으로 돌아온 뒤 삭제합니다.
    ```bash
    git checkout main
    git pull origin main
    git branch -d 브랜치이름
    ```

---

### 💡 실수했을 때 대처법

**"코드를 고치다가 처음으로 되돌리고 싶어요"**
```bash
git checkout -- .
```
(커밋하지 않은 모든 변경 사항이 사라지고 마지막 저장 상태로 돌아갑니다. 주의해서 사용하세요!)

**"작업하던 내용을 잠시 보관하고 다른 일을 해야 해요"**
```bash
git stash
```
(현재 작업을 임시 보관함에 숨겨둡니다. 나중에 `git stash pop`으로 다시 꺼낼 수 있어요.)

**"충돌(Conflict)이 발생했어요!"**
여러 사람이 같은 파일을 동시에 고치면 발생할 수 있습니다. 당황하지 말고 VS Code에서 `<<<< HEAD` 같은 표시가 있는 부분을 찾아 남길 코드를 선택하고 다시 커밋해 주세요. 해결이 어렵다면 팀장님께 도움을 요청하세요!
