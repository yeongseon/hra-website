# 보고서 제출 안내

본 문서는 HRA 보고서의 **PDF 변환 → 파일명 규칙 → 제출**까지의 절차를 안내합니다.

## 1. 전체 흐름

1. [보고서 양식](/member/templates)에서 분야별 양식 선택
2. Markdown 본문 작성 (참고: [Markdown 문법 가이드](/member/guides/markdown-guide))
3. 조원 검토 및 지도교수 컨펌
4. 사이트에서 **PDF 다운로드**
5. 파일명 규칙에 맞게 파일명 변경
6. 지정된 경로로 제출

## 2. PDF 다운로드 방법

1. [보고서 양식 목록](/member/templates) 페이지 접속
2. 작성한 분야의 양식 상세 페이지 진입
3. 우측 상단 **"PDF로 저장"** 버튼 클릭
4. 브라우저 인쇄 대화상자가 자동으로 열립니다.
5. **대상**을 `PDF로 저장`으로 선택 후 저장

> 인쇄 시 별도 설정은 필요 없습니다. A4·여백·글꼴은 시스템에서 자동 적용됩니다.

## 3. 파일명 규칙

PDF 파일명은 다음 형식을 따릅니다.

```
HRA{기수}_{조}_{분야}_{주제}.pdf
```

### 가. 분야 코드

| 분야 | 코드 |
| --- | --- |
| 경영서 | `경영서` |
| 고전명작 | `고전명작` |
| 기업실무·한국경제사 | `기업실무` |

### 나. 작성 예시

| 상황 | 파일명 |
| --- | --- |
| 19기 1조, 경영서, 『기업가정신』 | `HRA19_1조_경영서_기업가정신.pdf` |
| 19기 3조, 고전명작, 『논어』 | `HRA19_3조_고전명작_논어.pdf` |
| 20기 2조, 기업실무, 삼성전자 사례 | `HRA20_2조_기업실무_삼성전자사례.pdf` |

### 다. 작성 규칙

- 기수 번호는 숫자만 (`19`, `20`)
- 조 번호 뒤에는 반드시 `조` 표기 (`1조`, `2조`)
- 분야 코드는 위 표 그대로
- 주제는 공백 없이 작성하거나 언더스코어(`_`) 대신 사용 금지
- 특수문자(`/ \ : * ? " < > |`) 사용 금지

## 4. 제출 경로

| 제출 항목 | 경로 |
| --- | --- |
| PDF 본 | 지도교수 이메일 + 운영팀 공용 드라이브 |
| Markdown 원본 | 운영팀 공용 드라이브 (백업용) |
| 발표 자료 (선택) | 별도 슬라이드 파일 첨부 가능 |

> 정확한 제출 이메일·드라이브 주소는 매 학기 시작 시 운영팀에서 공지합니다.

## 5. 제출 마감

- **1차 검토본**: 발표 1주 전까지 지도교수 송부
- **최종본**: 발표일 당일 오전까지 운영팀 송부
- 마감 후 제출은 불가하며, 부득이한 경우 운영팀에 사전 연락 필수

## 6. 검토 체크리스트

- [ ] 발표 정보 표 모든 항목 작성 완료
- [ ] 5장 목차 누락 없음
- [ ] 출처·참고자료 명시
- [ ] 맞춤법 검사 완료
- [ ] PDF 변환 결과 깨짐 없음 확인
- [ ] 파일명 규칙 준수
- [ ] 지도교수 컨펌 완료

## 7. 자주 묻는 질문

**Q. PDF에 여백이 너무 좁거나 넓게 나옵니다.**
브라우저의 인쇄 미리보기에서 "여백: 기본"을 선택하세요. 시스템에서 A4 표준 여백을 적용합니다.

**Q. 한글 글꼴이 깨집니다.**
사이트 PDF 출력은 Pretendard / Noto Sans KR을 사용합니다. 브라우저 인쇄 시 "배경 그래픽" 옵션을 켜둔 상태로 저장하면 글꼴이 정상 적용됩니다.

**Q. 표가 한 페이지에 안 들어갑니다.**
표가 너무 길면 두 개로 분할하거나, 셀 안의 긴 텍스트는 줄바꿈(`<br>`)으로 정리하세요.

**Q. 이미지가 PDF에 누락됩니다.**
허용 출처(Vercel Blob, 동일 출처)에서 불러온 이미지만 PDF에 포함됩니다. 외부 도메인 이미지는 자료실에 업로드 후 사용하세요.

**Q. 제출 후 수정이 필요합니다.**
운영팀에 즉시 연락하여 재제출 가능 여부를 확인하세요. 발표 후 수정은 원칙적으로 불가합니다.
