    HRA 웹사이트 데이터베이스 가이드¶

    ⭐ 데이터베이스가 뭔가요?

    쉽게 말하면 아주 똑똑한 엑셀 파일이에요. 회원 정보, 공지사항, 지원서 같은 데이터를 안전하게 저장하는 창고예요. 엑셀처럼 표(시트) 형태로 데이터가 쌓여요.

    이 문서는 "엑셀은 써봤다" 수준이면 충분히 따라할 수 있어요!

    1. 엑셀이랑 뭐가 비슷한가요?¶

    데이터베이스는 엑셀이랑 거의 같은 개념이에요. 이름만 다를 뿐이에요:

    엑셀에서 부르는 이름 데이터베이스에서 부르는 이름 예를 들면
    시트 테이블 "회원" 시트, "공지사항" 시트처럼 종류별로 나눠요
    한 줄(행) 행(Row) 회원 한 명의 정보, 공지사항 한 건
    열 제목 컬럼(Column) 이름, 이메일, 역할 같은 항목

    💡 "테이블에 행을 추가한다" = "엑셀 시트에 한 줄 더 적는다" 라고 생각하면 돼요!

    2. ⭐ 우리 데이터베이스는 어디 있나요? (Neon)¶

    우리 사이트의 데이터는 Neon (neon.tech)이라는 무료 온라인 서비스에 저장돼요. 인터넷 브라우저에서 바로 접속해서 데이터를 볼 수 있어요.

    접속 방법 (3단계)¶

    1. 브라우저에서 neon.tech에 로그인해요
    2. 왼쪽에서 우리 프로젝트를 클릭해요 (예: hra-website)
    3. 왼쪽 메뉴에서 SQL Editor를 클릭해요

    👉 여기에 명령어를 입력하고 Run 버튼을 누르면 데이터를 보거나 수정할 수 있어요!

    엑셀처럼 보는 방법¶

    왼쪽 메뉴에서 Tables 탭을 누르면 엑셀처럼 표 형태로 데이터를 바로 볼 수 있어요. SQL 명령어를 몰라도 괜찮아요!

    3. 우리 사이트에는 어떤 데이터가 저장되나요?¶

    ① 회원 정보 (users 테이블)¶

    소셜 로그인(구글/카카오)이나 관리자 계정 생성 스크립트로 회원이 등록돼요.

    항목 뭘 저장하나요? 예시
    이름 로그인할 때 가져온 이름 "홍길동"
    이메일 로그인에 사용한 이메일 "hong@gmail.com"
    비밀번호 해시 관리자 계정의 비밀번호가 암호화되어 저장됨 password_hash
    역할 일반 회원인지 관리자인지 ADMIN(관리자) 또는 MEMBER(일반 회원)
    프로필 사진 소셜 로그인에서 가져온 사진 (자동)
    가입 날짜 처음 로그인한 날 2025-03-01

    🔒 이 페이지는 누가 볼 수 있나요? (권한 안내)¶

    회원의 '역할'에 따라 들어갈 수 있는 페이지가 자동으로 제한돼요. 로그인을 안 했거나 권한이 없으면 로그인 페이지나 메인으로 이동해요.

    페이지 주소 필요한 역할 설명
    /admin/* ADMIN (관리자) 관리자 전용 메뉴 (회원 관리, 설정 등)
    /resources/* 로그인 필요 (누구나) 자료실 이용
    /mypage/* 로그인 필요 (누구나) 내 정보 확인 및 수정
    그 외 모든 페이지 제한 없음 (공개) 누구나 볼 수 있는 일반 페이지

    💡 중요: 사용자의 역할을 MEMBER에서 ADMIN으로 바꾸면, 즉시 관리자 전용 페이지에 접속할 수 있게 돼요! (반대의 경우도 마찬가지예요)

    ② 기수 정보 (cohorts 테이블)¶

    각 기수(1기, 2기...)의 정보를 저장해요.

    항목 뭘 저장하나요? 예시
    기수 이름 몇 기인지 "3기"
    모집 상태 모집 중인지 마감인지 모집중 / 마감 / 예정
    활동 기간 시작일 ~ 종료일 2025-03-01 ~ 2025-08-31

    ③ 지원서 (applications 테이블)¶

    학생들이 사이트에서 제출한 지원서예요.

    항목 뭘 저장하나요? 예시
    지원 기수 어느 기수에 지원했는지 3기
    지원자 이름 지원한 사람 이름 "김학생"
    지원자 이메일 연락처 이메일 "kim@naver.com"
    지원 동기 왜 지원했는지 "성장하고 싶어서..."
    처리 상태 대기/합격/불합격 대기 → 합격 or 불합격

    💡 관리자 페이지의 "지원서" 메뉴에서 합격/불합격을 바꿀 수 있어요!

    ④ 공지사항 & 자료실¶

    종류 저장 방식 설명
    공지사항 GitHub CMS content/notices/ 폴더에 마크다운 파일로 저장돼요.
    자료실 DB (resources) 수업일지를 포함한 각종 자료가 데이터베이스에 저장돼요.

    💡 공지사항은 GitHub을 통해 관리되므로 데이터베이스 SELECT로는 나오지 않아요! 관리자 페이지에서 수정하면 자동으로 GitHub에 반영됩니다.

    ⑤ 갤러리 (활동 사진)¶

    하나의 갤러리(앨범)에 여러 장의 사진을 올릴 수 있어요. 사진첩이라고 생각하면 돼요!

    4. ⭐ 관리자 계정 만들기¶

    왜 필요한가요?¶

    사이트를 처음 만들면 관리자가 아무도 없어요. 누군가가 "이 사람은 관리자야"라고 직접 지정해줘야 해요.

    방법 1: 스크립트로 만들기 (가장 쉬움) ⭐¶

    터미널에서 아래 명령어를 입력하세요:

    ADMIN_EMAIL="admin@hra.ac.kr" ADMIN_PASSWORD="원하는비밀번호" npm run seed-admin
    • 이메일과 비밀번호를 원하는 값으로 바꿔서 입력하세요
    • 이미 존재하는 이메일이면 해당 계정을 관리자로 승격시켜줘요
    • 생성 후 로그인 페이지에서 이메일/비밀번호로 바로 로그인!

    방법 2: 소셜 로그인 후 SQL로 승격하기¶

    1단계: 먼저 사이트에서 구글이나 카카오로 로그인해요.

    👉 처음 로그인하면 자동으로 일반 회원(MEMBER) 계정이 만들어져요.

    2단계: neon.tech에 로그인 → 왼쪽에서 프로젝트 선택 → SQL Editor 클릭 → 아래 명령어를 복사 붙여넣기 → Run 버튼 클릭

    UPDATE users SET role = 'ADMIN' WHERE email = '내이메일@example.com';
    • 내이메일@example.com 부분을 본인이 로그인할 때 쓴 이메일로 바꿔야 해요!
    • 작은따옴표(')는 지우면 안 돼요!

    3단계: 사이트에서 로그아웃 → 다시 로그인 → 상단에 "관리자" 메뉴가 보이면 성공! 🎉

    더 쉬운 방법 — 관리자 페이지에서 바꾸기¶

    첫 번째 관리자가 만들어진 후에는 SQL을 안 써도 돼요!

    1. 관리자 페이지 → 회원 관리 메뉴를 클릭해요
    2. 바꾸고 싶은 사람 옆에 "관리자로 변경" 또는 "멤버로 변경" 버튼이 있어요
    3. 버튼을 누르면 바로 반영돼요!

    💡 자기 자신의 역할은 바꿀 수 없어요. 실수로 관리자 권한을 빼는 걸 막으려고 일부러 그렇게 만들었어요!

    5. 회원이나 관리자를 삭제하고 싶으면?¶

    방법 1: 관리자 권한만 없애기¶

    관리자 페이지 → 회원 관리 → "멤버로 변경" 버튼

    방법 2: 계정 자체를 완전히 삭제하기¶

    Neon SQL Editor에서 아래를 실행해요.

    ⚠️ 주의: 삭제하면 되돌릴 수 없어요! 먼저 누가 있는지 확인하세요.

    -- 먼저 회원 목록을 확인해요 (이건 보기만 하는 거라 안전해요)
SELECT id, name, email, role FROM users;

    👉 위 명령어를 실행하면 모든 회원이 표로 나와요. 삭제할 사람의 이메일을 확인하세요.

    -- 회원 삭제하기
DELETE FROM users WHERE email = '삭제할이메일@example.com';

    ⚠️ 그 사람이 쓴 자료실 글이 있으면, 글을 먼저 지우고 나서 회원을 삭제해야 해요:

    -- 1. 그 사람이 쓴 자료실 글 삭제
DELETE FROM resources WHERE author_id = '그사람의_ID값';

-- 2. 마지막으로 회원 삭제
DELETE FROM users WHERE email = '삭제할이메일@example.com';

    👉 그사람의_ID값은 위에서 SELECT 했을 때 나온 id 열의 값이에요. 👉 공지사항은 GitHub에 저장되므로 별도로 삭제하지 않아도 회원 삭제에 영향을 주지 않아요. (하지만 글은 남게 됩니다)

    ⚠️ 관리자를 전부 삭제하면 관리자 페이지에 아무도 못 들어가요! 최소 1명은 남겨두세요.

    6. 이런 문제가 생기면 이렇게 하세요¶

    이런 상황이에요 이렇게 하면 돼요
    SQL을 실행했는데 "0 rows affected"라고 나와요 이메일을 잘못 적은 거예요. SELECT email FROM users;로 정확한 이메일을 확인하세요
    역할을 바꿨는데 사이트에서 안 바뀌어요 사이트에서 로그아웃했다가 다시 로그인해보세요
    "foreign key constraint" 에러가 나요 그 사람이 쓴 글이 있어서 그래요. 위의 삭제 순서(글 먼저 → 회원 나중에)를 따라하세요

    7. Neon에서 자주 쓰는 명령어 모음¶

    아래 명령어들을 SQL Editor에 복사 붙여넣기하고 Run을 누르면 돼요.

    -- 📋 모든 회원 보기
SELECT id, name, email, role FROM users;

-- 👑 관리자만 보기
SELECT name, email FROM users WHERE role = 'ADMIN';

-- 📚 자료실 최근 글 10개 보기
SELECT title, category, created_at FROM resources ORDER BY created_at DESC LIMIT 10;

-- 🔢 관리자가 몇 명인지 세기
SELECT COUNT(*) FROM users WHERE role = 'ADMIN';

-- 📬 지원서 현황 보기
SELECT applicant_name, applicant_email, status FROM applications;

-- 📬 특정 기수(예: 3기)의 지원서만 보기
SELECT a.applicant_name, a.status, c.name as cohort_name
FROM applications a
JOIN cohorts c ON a.cohort_id = c.id
WHERE c.name = '3기';

    💡 꿀팁: 데이터를 수정하거나 삭제하기 전에 항상 SELECT(보기)부터 먼저 해보세요. 실수로 중요한 데이터를 날리면 되돌리기 어려워요!

    8. 데이터 항목(컬럼)을 추가하고 싶으면?¶

    예를 들어 "전화번호" 항목이 더 필요하다면:

    1. VS Code에서 src/lib/db/schema.ts 파일을 열어요
    2. 원하는 테이블에 새 항목을 추가해요 (잘 모르겠으면 관리자에게 물어보세요!)
    3. 터미널(검은 창)에서 아래 명령어를 입력해요
    npx drizzle-kit push
    1. "Changes applied"라고 나오면 성공! 실제 데이터베이스에 반영된 거예요.

    ⚠️ 주의: 이미 있는 항목을 지우거나 종류를 바꾸면 기존 데이터가 사라질 수 있어요! 잘 모르겠으면 꼭 관리자에게 물어보세요.
